cadastro e edição de perfis de webservice. \n\nOpen image-20250620-165253.png\n\nPara criar um novo perfil clique no botão 'Inserir Novo Perfil'\n\nNo primeiro passo do assistente temos os seguintes dados:\n\nCódigo - Código auto numerável atribuído pelo banco de dados (bloqueado para edição)\n\nAtivo(SIM/NÃO) - Define se a regra está ativa\n\nData/Hora - Data e hora da criação do perfil (bloqueado para edição)\n\nOperador - Operador que criou o perfil (bloqueado para edição)\n\nTempo de Expiração - Tempo em minutos de expiração do token de autenticação.\n\nInvalidar(SIM/NÃO) - \n\nMarcando Sim neste campo, o token será invalidado a partir da sua utilização ou \n\nCaso contrário vai respeitar o tempo em minutos informado para ser considerado inválido.\n\nContra-senha - \n\nContra-senha requisitada para a autenticação gerar o token de callback. \n\nO sistema irá sugerir uma contra-senha\n\nA contra senha deve ter no mínimo 8 dígitos entre números e caracteres.\n\nDescrição identificativa do perfil - Informe a descrição que irá identificar o perfil cadastrado em seu sistema.\n\n\n\n\nAtenção\n\n A liberação deste perfil é de sua inteira responsabilidade, assim como o consumo dos serviços a ele atrelados \n\nClique no botão 'Próximo' para seguir para o passo seguinte\n\nOpen image2019-9-27_10-53-15.png\n\nNeste segundo passo, o operador poderá restringir a utilização dos webservices para determinados endereços de IPs.  \n\nClique no botão 'Adicionar IP à relação'\n\nOpen opção 'Limitar consumo de serviços a lista de IPs' para obrigar a inserção de IPs ao perfil e para seja feita a restrição de acesso ao webservices. \n\nClique no botão 'Próximo' para seguir para o passo seguinte\n\nOpen image2019-9-27_10-53-50.png\n\nNesse terceiro passo o operador poderá restringir a utilização dos Webservices por usuário do sistema.\n\nPasse para o quadro direito os usuário
