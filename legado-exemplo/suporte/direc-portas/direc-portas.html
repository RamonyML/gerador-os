Direcionamento de portas ou Port Forwarding


É o processo de configurar um roteador para permitir que tráfego de rede externo acesse dispositivos ou serviços específicos na rede local (LAN). Isso é essencial para permitir o acesso remoto a dispositivos como câmeras de segurança, servidores de jogos, ou outros serviços que precisam estar acessíveis fora da rede local.

Como Funciona o Direcionamento de Portas?


Identificação da Porta e do IP Interno: Cada dispositivo ou serviço na rede local utiliza portas específicas (portas TCP/UDP) para comunicação. Primeiro, você precisa identificar quais portas são necessárias para o serviço (por exemplo, portas utilizadas pelo DVR das câmeras de segurança). Também é necessário saber o endereço IP interno do dispositivo (por exemplo, o DVR) para onde o tráfego deve ser direcionado.

Configuração no Roteador:
Acesse a interface da Firmware do roteador/ONT. Digitando o endereço IP do roteador (algo como 192.168.0.1 ou 192.168.1.1).

Criação de regras de direcionamento de portas:
Localize a seção de Port Forwarding (direcionamento/apontamento/encaminhamento de portas), Virtual Server, NAT ou similar.
• Crie uma nova regra de direcionamento, especificando a porta ou o intervalo de portas que precisa ser redirecionado.
• Defina o protocolo (TCP, UDP ou ambos).
• Insira o endereço IP interno do dispositivo que receberá o tráfego (o DVR, Computador, ou dispositivo em questão no caso).
• Salve as configurações.



Exemplo Prático com DVR


Se o DVR das câmeras de segurança utiliza a porta 554 para acesso, e o IP interno do DVR é 192.168.1.10: - Porta Externa: 554 - Porta Interna: 554 - Protocolo: TCP/UDP - Endereço IP Interno: 192.168.1.10 *Por que é Necessário?* - *Acesso Remoto*: Permite que usuários acessem dispositivos ou serviços da rede local através da internet. - *Segurança*: Ao especificar quais portas podem ser acessadas externamente, você pode limitar a exposição da rede a possíveis ameaças. > Considerações sobre CGNAT e IP Público *CGNAT (Carrier-Grade NAT)*: - CGNAT é uma técnica usada por provedores de internet para gerenciar a escassez de endereços IPv4. Com CGNAT, múltiplos clientes compartilham um único endereço IP público. - Se a conexão do cliente estiver dentro da rede CGNAT pode dificultar o direcionamento de portas porque o roteador não possui um IP público único e dedicado. Isso impede que tráfego externo seja roteado corretamente para a rede local do cliente. *IP Público Fora CGNAT*: - Para que o direcionamento de portas (exemplificado acima) funcione corretamente, é ideal que o cliente possua um IP público, fora do CGNAT. - Com um IP público, o roteador pode receber diretamente o tráfego externo e aplicar as regras do direcionamento de portas para direcionar o tráfego ao dispositivo correto na rede local. *Procedimentos Adicionais*: 1. *Solicitação de IP Público*: O cliente pode solicitar a alteração do plano de internet com IP público fora CGNAT. 2. *Configuração do Roteador*: Após receber um IP público, configure o roteador do cliente para usar esse IP. O procedimento exato pode variar dependendo do roteador e do provedor de internet. Ao configurar o direcionamento de portas com um IP público, os dispositivos na rede local poderão ser acessados de forma remota, conforme as regras estabelecidas.
