Extensão "Registros" — Firebase / Firestore

==========================================



1) Domínio autorizado (Authentication → Settings → Authorized domains)

   Adicione exatamente (troque pelo ID da sua extensão em chrome://extensions):



   chrome-extension://SUA_EXTENSAO_ID



   O ID aparece na página da extensão carregada em modo desenvolvedor.



   --- Sobre o ID em "Carregar sem compactação" (modo desenvolvedor) ---

   Não é "aleatório" a cada abertura: o Chrome calcula o ID a partir do

   pacote da extensão (em modo unpacked, isso depende da pasta/caminho e do

   conteúdo). Por isso:

   - Dois computadores ou duas pastas diferentes → IDs diferentes são normais.

   - Na mesma máquina e mesma pasta, o ID costuma ser estável.



   O que fazer:

   - Desenvolvimento: você pode cadastrar VÁRIOS domínios autorizados no Firebase

     (um chrome-extension://... por ID que realmente usar).

   - Produção: ao publicar na Chrome Web Store, a extensão ganha um ID fixo;

     use esse na lista e mantenha os de dev só enquanto precisar.

   - Opcional (avançado): campo "key" no manifest.json com uma chave fixa faz

     o ID permanecer o mesmo ao empacotar/carregar a mesma base em várias máquinas;

     só vale a pena se todo mundo usar exatamente o mesmo fluxo/pasta empacotada.



2) Firestore → Rules

   NÃO substitua o arquivo inteiro de regras do projeto se já existirem match para

   users, operadores, upgrades, logs etc. Adicione APENAS este bloco dentro de

   match /databases/{database}/documents { ... } e publique:



   match /mensagensRapidas/{userId}/itens/{itemId} {

     allow read, write: if request.auth != null && request.auth.uid == userId;

   }



   (O trecho abaixo era só um exemplo "isolado"; na prática sempre faça merge.)



   rules_version = '2';

   service cloud.firestore {

     match /databases/{database}/documents {

       match /mensagensRapidas/{userId}/itens/{itemId} {

         allow read, write: if request.auth != null && request.auth.uid == userId;

       }

     }

   }



3) Coleção usada pela extensão

   mensagensRapidas/{uid do operador}/itens/{documento}

   Campos: title (string), message (string), createdAt, updatedAt (timestamp)
