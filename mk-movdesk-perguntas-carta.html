1 Contexto

Estamos integrando o sistema interno gerador-os / mznet.digital com o MK Solutions ERP via webservices. O objetivo é automatizar a abertura de atendimentos e criação de Ordens de Serviço para as demandas do setor de suporte técnico.

Temos 186 variantes mapeadas em 8 categorias. Atualmente WSMKNovoAtendimento está funcionando em produção (atendimento #268168 validado). No entanto, dois endpoints estão bloqueando o avanço da maior parte das variantes:

EndpointCategorias impactadasVariantes travadasStatus
WSMKCriarOrdemServico.rule Manutenção, Altplan com visita, Wi-Fi Extend, Mud. Endereço ~135 ⚠ Aguardando
WSMKAtendimentoComentario.rule Feedback de visita, Encerramentos de altplan 11 ✗ HTTP 500

2 Perguntas

  • Q1
    WSMKCriarOrdemServico.rule — está incluso no nosso pacote ou é API especial? Precisamos criar Ordens de Serviço técnicas diretamente via API. A criação manual no painel não se encaixa no fluxo integrado porque o atendimento é gerado automaticamente pelo sistema e a O.S. deve ser vinculada ao mesmo protocolo em tempo real.
    Manutenção — 77 variantes Altplan com visita — 38 variantes Wi-Fi Extend — 10 variantes Mudança de endereço — 10 variantes
    Se for API especial: qual o nome comercial do módulo e como contratamos?
    Se já estiver incluso: precisamos dos códigos CodigoTipoOS, CodigoGrupoServico e confirmar se CodigoTecnico pode ser omitido na abertura (técnico designado depois pelo painel).
  • Q2
    WSMKAtendimentoComentario.rule — bug na instalação ou módulo não contratado? Em chamado anterior o analista Jardel Henrique informou que "a API não está na nossa base". Estamos retornando HTTP 500 no POST do comentário. Precisamos confirmar: esse endpoint requer a contratação de um módulo separado?
    Feedback de visita — 8 variantes Encerramentos de altplan — 3 variantes
    O fluxo precisa: 1) localizar o atendimento aberto, 2) inserir comentário de encerramento com texto padronizado. Sem esse endpoint o feedback técnico pós-visita não pode ser automatizado.

    Se for módulo a contratar: qual o nome e valor? Se for configuração: qual o passo para ativar?
  • Q3
    Existe endpoint para buscar um atendimento existente por número de protocolo? O fluxo de feedback precisa localizar um atendimento já aberto para então inserir o comentário de encerramento (via WSMKAtendimentoComentario). Atualmente conseguimos buscar o cliente por CPF/CNPJ, mas não encontramos endpoint para buscar um atendimento/protocolo por número.
    Feedback de visita — 8 variantes
    Existe WSMKBuscarAtendimento.rule ou similar? Se não houver, há alguma outra forma de obter o CodigoAtendimento a partir do número de protocolo visível para o operador?
